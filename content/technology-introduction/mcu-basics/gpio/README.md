# GPIO

`GPIO` 是 MCU 学习里最基础、也最常用的一部分。理解 GPIO，不只是为了点亮一个 `LED`，更重要的是理解“一个引脚是如何在输入、输出、复用和模拟之间切换的”。

## 1. 这是什么

`GPIO` 是 `General Purpose Input Output` 的缩写，通常翻译为“通用输入输出口”。

它最基础的作用有两种：

- 作为输入口读取外部电平
- 作为输出口向外部输出高低电平

在 `STM32` 里，一个引脚往往不只具有 GPIO 这一种用途，它还可能被复用为串口、定时器、`SPI`、`I2C`、`ADC` 等外设功能。所以可以把 GPIO 理解成“引脚最基础、最通用的工作方式”。

## 2. 结构图与基本结构

下面这张图适合先建立整体印象。读图时优先关注 4 个位置：输入通路、上拉/下拉网络、`MUX` 选择，以及输出驱动级。

![GPIO 基本结构示意](../images/02-gpio-basic.png)

GPIO 结构里最值得优先理解的是下面几部分：

| 模块 | 基本作用 | 直观理解 | 说明 |
|---|---|---|---|
| 保护二极管 | 把引脚电压限制在安全范围附近 | 电压异常时先泄放一部分电流 | 主要用于过压和静电保护，不是正常功能电路 |
| 施密特触发器 | 把不够干净的输入边沿整理成稳定数字电平 | 带迟滞的整形门限 | 对抗毛刺和抖动很重要 |
| 输入缓冲器 | 把引脚上的外部电平送入内部逻辑 | 把外部信号“读进来” | 数字输入模式下会经过这里 |
| 上拉/下拉网络 | 给输入引脚提供默认电平 | 轻轻把引脚拉高或拉低 | 主要为了避免悬空 |
| MUX | 在多条内部信号路径中切换 | 像一个切换开关 | 决定引脚当前由 GPIO 还是外设接管 |
| 输出驱动级 | 把内部控制结果真正送到引脚 | 把内部命令“推到引脚” | 推挽和开漏的核心差别就在这里 |

可以先抓住一句话：输入时重点看“怎么读得稳”，输出时重点看“怎么推得出去”，复用时重点看“这个引脚现在归谁管”。

## 3. 常见模式

`STM32F1` 的 GPIO 常见模式可以先按“输入 / 输出 / 复用 / 模拟”来理解：

| 模式 | 类别 | 基本含义 | 关键特点 | 常见用途 |
|---|---|---|---|---|
| Analog input | 输入 | 模拟输入 | 关闭数字输入缓冲，减少干扰 | `ADC` 输入 |
| Floating input | 输入 | 浮空输入 | 内部不上拉也不下拉 | 外部已提供稳定电平的输入 |
| Input with pull-up / pull-down | 输入 | 上拉/下拉输入 | 给引脚一个默认状态，避免悬空 | 按键、开关量输入 |
| General purpose output push-pull | 输出 | 通用推挽输出 | 高低电平都能主动输出 | LED、普通数字输出 |
| General purpose output open-drain | 输出 | 通用开漏输出 | 只能主动拉低，高电平依赖上拉 | `I2C`、共线控制 |
| Alternate function output push-pull | 复用输出 | 复用功能推挽输出 | 本质仍是推挽，但由外设控制 | `USART`、`SPI`、定时器输出 |
| Alternate function output open-drain | 复用输出 | 复用功能开漏输出 | 本质仍是开漏，但由外设控制 | `I2C` 等需要开漏的外设 |
| Alternate function input | 复用输入 | 复用功能输入 | 引脚作为外设输入通道使用 | 串口接收、输入捕获等 |

## 4. 关键概念

### 4.1 上拉 / 下拉

- 上拉：默认偏高
- 下拉：默认偏低
- 作用：给输入口一个默认状态，避免悬空

### 4.2 推挽 / 开漏

- 推挽输出：高低电平都能主动输出，驱动能力更强
- 开漏输出：只能主动拉低，输出高电平要依赖上拉电阻

### 4.3 普通 GPIO / 复用功能

- 普通 GPIO：由软件直接控制读写电平
- 复用功能：引脚交给外设控制，例如串口、定时器、SPI、I2C

## 5. 看图时怎么抓重点

| 关注点 | 读图时看哪里 | 先理解什么 |
|---|---|---|
| 输入稳定性 | 输入缓冲器、施密特触发器、上拉/下拉网络 | 输入是否会悬空，边沿是否容易抖动 |
| 功能切换 | `MUX` | 这个引脚当前是普通 GPIO 还是已经交给外设 |
| 输出能力 | 输出驱动级 | 是推挽还是开漏，能不能主动拉高 |
| 模拟信号 | 模拟通路 | 为什么模拟输入要尽量避开数字通路干扰 |

## 6. 工作过程

可以把 GPIO 的工作理解成下面这条链路：

```mermaid
flowchart LR
    A["外部引脚电平"] --> B["保护电路"]
    B --> C["输入通路 / 输出驱动级"]
    C --> D["MUX 选择"]
    D --> E["GPIO 逻辑 或 外设逻辑"]
    E --> F["寄存器 / 程序控制"]
```

学习时最关键的是知道：

1. 输入时，重点在于是否悬空、是否需要上拉下拉
2. 输出时，重点在于推挽和开漏
3. 复用时，重点在于这个引脚是不是已经交给外设了

## 7. 常见用途

| 用途 | 说明 |
|---|---|
| 点亮 LED | 最基础的输出控制练习 |
| 读取按键 | 最基础的输入练习 |
| 软件模拟时序 | 例如简单控制 OLED、位操作协议 |
| 外设引脚复用 | 给 `USART`、`SPI`、`I2C`、定时器等外设提供引脚通路 |

## 8. 常见错误

| 问题 | 说明 |
|---|---|
| 输入悬空 | 没有上拉/下拉时，输入电平会不稳定 |
| 把开漏当推挽用 | 开漏不能主动输出高电平 |
| 忘记开 GPIO 时钟 | 配置前必须先使能对应端口时钟 |
| 复用冲突 | 同一引脚可能已被调试口或外设占用 |
| 板载 LED 逻辑相反 | 很多开发板是低电平点亮，不是高电平点亮 |

## 9. 结合当前 STM32 项目理解

放到当前项目里，可以这样理解：

| 引脚 | 当前用途 | 对应理解 | 说明 |
|---|---|---|---|
| `PC13` | 板载 LED | 普通输出模式 | 适合练习 GPIO 最基础的输出控制 |
| `PB8 / PB9` | OLED 相关连线 | GPIO 与软件时序 | 适合理解拉高拉低与时序控制的关系 |
| `PA13 / PA14` | 下载与调试接口 | 复用占用 | 常被调试功能占用，不适合作为入门练习对象 |
