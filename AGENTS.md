# Project Working Rules

## Startup

Before substantial changes:

1. Read `README.md`
2. Read `memory/project-overview.md`
3. Read `memory/decisions.md`
4. Read `notes/todo.md`

## Documentation Discipline

- This project uses `structure_profile: docs-root-knowledge`. Keep `docs/` as the canonical documentation root.
- Keep `workspace.yaml` aligned with the real project status and entry points.
- Record durable decisions in `memory/decisions.md`.
- When the user asks for a weekly report, update `notes/weekly-summary/YYYY-Www.md` using the project weekly template and only record real work from that week.

## Editing Boundaries

- Preserve `docs/`, `content/`, `templates/`, and `tools/` as the main top-level structure.
- Do not reintroduce the old stm32 resource path assumptions.
