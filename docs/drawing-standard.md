# 资料页与 SVG 绘制规范

适用范围：
- 器件资料页 `README.md`
- `pinout / internal / application` 图示
- 技术介绍页中的结构图、流程图、时序图
- 后续自绘 `SVG` 的统一布局与样式

目标：
- 查阅优先
- 结构统一
- 适合本地 Markdown 与 GitHub Pages

## 1. 单器件资料页结构

每个器件页统一按下面顺序组织：

1. 标题
2. 来源
3. `Pin 图与引脚说明`
4. `基本参数`
5. `使用方式`
6. `备注`

如有额外图片，再补：

7. `内部原理图`
8. `实际案例图`

要求：
- 默认以表格为主
- 不写大段说明文字
- 器件页用于“快速查阅”，不是长篇科普

## 2. 目录结构

一个器件一个目录：

```text
content/device-principles/<category>/<component-id>/
├─ README.md
└─ images/
   ├─ pinout/
   ├─ internal/
   └─ application/
```

要求：
- 正文固定放在 `README.md`
- 图片按用途分类
- 全部使用相对路径
- 目录名优先使用“器件类型”或“特定型号”
- 不用公司名或系列名作为主目录名，厂家信息只放在正文来源或备注中

命名建议：

- 类型名：`aluminum-electrolytic-capacitor`、`ceramic-capacitor`
- 型号名：`ne555p`、`stm32f103c8t6`

## 3. 图片来源原则

优先级：

1. 厂家或来源网页已有清晰图片时，优先引用或下载使用
2. 没有合适单图时，再自绘 `SVG`
3. 自绘 `SVG` 必须遵循本文档规范

## 4. SVG 总体布局

推荐统一采用“三栏式纵向结构”：

1. `标题栏`
2. `模型栏`
3. `标注说明栏`

推荐比例：

- 标题栏：整体高度的 `12%` 到 `15%`
- 模型栏：整体高度的 `50%` 到 `58%`
- 标注说明栏：整体高度的 `28%` 到 `35%`

推荐画布：

- 横向画布
- `viewBox` 建议 `920 x 520` 附近
- 四周保留明显留白

要求：
- 标题不能压到模型
- 标注不能压到模型
- 说明框不能超出边框
- 同一张图的布局应稳定，不随内容随意漂移

补充：
- 对技术主题图，`模型栏` 可以理解为“主体示意区”，其中可放结构链路、波形、时序或信号关系
- 如果一张图同时包含“原理区”和“波形区”，优先让说明文字进入底部独立说明栏，而不是悬浮压在波形上

## 5. 标题栏规范

作用：
- 只放主标题

要求：
- 标题必须完全位于顶部安全区内
- 标题与模型之间至少保留一行以上空白
- 不在标题栏内放复杂说明

推荐：
- 左对齐
- 标题字号 `22px`

## 6. 模型栏规范

作用：
- 放器件主体
- 放主要尺寸标注
- 放关键外观识别标注

模型栏建议再分成三块：

- 左侧：尺寸、长短脚、方向
- 中间：器件主体
- 右侧：极性、印字、壳体识别

要求：
- 主体必须居中稳定
- 左右说明线尽量短
- 不允许文字跨越器件主体
- 不允许多条箭头线大面积交叉

## 7. 标注说明栏规范

作用：
- 放速读总结
- 放“如何看这个器件”

要求：
- 统一放底部
- 使用一个较小的说明框
- 内容控制在 `2` 到 `4` 条
- 文本必须完全落在框内
- 如果说明较长，必须主动拆成两行或多行，不能依赖单行硬撑
- 说明框宽度不足时，优先增加说明栏高度，不要压缩主图区域
- 说明框不得遮挡波形、尺寸线、箭头线和关键标注

不建议：
- 在模型栏中塞过大的说明卡片
- 说明框抢占主图空间
- 在右下角放大段说明后覆盖主波形或主模型

## 8. 字体与字号

推荐层级：

- 标题：`22px`
- 一级标签：`18px`
- 普通说明：`15px`
- 小字说明：`13px`

要求：
- 同一张图最多使用 `4` 个字号层级
- 行距必须足够，不能粘连
- 文本不能贴线、压线、溢出
- 边框内文字必须预留左右内边距，不能贴着圆角边框
- 长句说明优先减字数，其次拆行，最后才考虑缩小字号

## 9. 颜色规范

推荐分层：

- 主体颜色：棕、灰、黑
- 正极提示：偏暖色
- 负极提示：偏深红
- 尺寸线：深灰
- 背景：浅暖色

要求：
- 颜色服务于信息分层
- 同类器件尽量复用配色
- 不堆叠复杂渐变和装饰

## 10. 元件表达规范

### 10.1 极性器件

必须明确表达：

- 是否区分正负极
- 如何从引脚判断
- 如何从壳体判断
- 如何读壳体上的数字或字母

### 10.2 无极性器件

必须明确表达：

- 无极性
- 两端是否可互换
- 常见标识如何阅读

### 10.3 引脚长度

如果器件存在长短脚差异：

- 图形上必须真实画出差异
- 文本说明必须和图形一致
- 不允许“图里长短相反、文字又相反”
- 如果在模型下方额外绘制 `legend` 或说明线，线长也必须与“长脚 / 短脚”的语义一致：
  - `长脚` 对应的说明线应明显更长
  - `短脚` 对应的说明线应明显更短
  - 不允许模型正确但 legend 反向

## 10.4 三栏布局校准要求

每次新增或修改 `SVG` 后，都要按下面顺序校准：

1. 检查标题是否完全位于标题栏内
2. 检查模型是否侵入标题栏
3. 检查底部说明框是否完全位于标注说明栏内
4. 检查所有文字是否在边框内，不能溢出
5. 检查有极性器件的正负极说明是否和模型一致
6. 检查长短脚说明、legend 长短、模型引脚长度三者是否一致

## 10.5 技术主题图额外校准要求

适用于 `PWM`、`GPIO`、`I2C`、`SPI`、`ADC` 这类技术介绍页配图。

每次新增或修改后，再额外检查：

1. 主示意区是否仍是最显眼区域，说明框不能喧宾夺主
2. 波形区、时序区、结构区是否被说明框遮挡
3. 关键公式、频率、周期、引脚名是否与正文一致
4. 如果有多个波形示例，波形与文字说明是否一一对应
5. 如果底部有总结框，内容是否控制在两行到三行内
6. 在 GitHub 页面缩放为常见桌面宽度时，文本是否仍可读

## 10.6 技术主题图推荐尺寸

对于“结构 + 波形 + 说明”类图，建议优先使用较高的画布：

- 推荐 `viewBox`：`1200 x 760` 到 `1200 x 860`
- 标题区：`10%` 到 `14%`
- 原理区：`30%` 到 `38%`
- 波形或时序区：`32%` 到 `40%`
- 底部说明区：`8%` 到 `14%`

这样更适合后续在 GitHub Pages 中直接嵌入查看。
