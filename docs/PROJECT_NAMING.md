# 项目命名与发布规范

本文用于统一本项目在本地、GitHub 仓库、GitHub Pages 站点中的命名方式。

## 1. 最终命名方案

本项目后续统一使用下面这组名称：

| 类型 | 名称 | 说明 |
|---|---|---|
| 仓库名 | `embedded-handbook` | 用于 GitHub 仓库名，短、稳定、便于公开发布 |
| 站点标题 | `嵌入式与器件手册` | 用于首页标题、Pages 页面标题、项目对外展示名 |
| 中文副标题 | `电子器件、MCU 基础与常用技术主题整理` | 用于首页简介、仓库描述扩展说明 |

## 2. 为什么采用这组名称

这套命名更适合当前项目的真实内容：

- 不只记录器件，还包括 `GPIO`、`PWM`、接口、原理说明
- 不只面向某一类芯片，后续还能继续扩展
- 既适合 GitHub 仓库名，也适合后续做成在线文档站点

## 3. 命名使用规则

### 3.1 仓库层

GitHub 仓库名固定建议为：

```text
embedded-handbook
```

不建议：

- 使用纯中文仓库名
- 使用过长仓库名
- 使用只覆盖某一部分内容的名字，例如只写 `components` 或只写 `stm32-notes`

### 3.2 页面展示层

Markdown 首页和后续站点首页统一使用：

```text
嵌入式与器件手册
```

副标题可使用：

```text
电子器件、MCU 基础与常用技术主题整理
```

### 3.3 目录与文档命名层

目录命名继续遵循现有规则：

- 目录名使用英文、小写、连字符
- 器件目录优先使用器件类型或具体型号
- 技术主题目录优先使用主题名，例如 `gpio`、`pwm`

## 4. 对外描述建议

用于 GitHub 仓库简介时，建议写成：

```text
嵌入式与器件手册：整理常用电子器件、MCU 基础知识与技术主题的可查阅资料库。
```

如果需要英文简介，可用：

```text
An embedded handbook for common electronic components, MCU basics, and practical technology notes.
```

## 5. 后续默认规则

以后如果需要：

- 改首页标题
- 补站点描述
- 增加 GitHub Pages 配置
- 统一仓库对外命名

默认都按本文执行。
